## Overview

Most CLI libraries ask you to learn a framework before you can write a single command: subclasses, parser configuration, help text wiring. For a small script, the boilerplate outweighs the logic.

Invokify takes a simpler approach. You decorate a function, feed a list of strings to a parser, and call the result. It's a small Python library that turns regular functions into commands you can invoke from a string or argument list, with nested subcommands when you need them.

## Commands from decorators

The core API is three steps: create an engine, decorate your functions, parse input.

```py
from invokify import InvokeEngine

engine = InvokeEngine()

@engine.command
def greet(user: str):
    print(f"Hello {user}!")

cmd, args, callstack = engine.parse(["greet", "Jeff"])
cmd(*args)
```

`parse` walks the argument list recursively, matching commands and subcommands until it hits a leaf. Whatever remains becomes the arguments passed to your function. The callstack records which commands were traversed, which helps when debugging nested command trees.

## Subcommands

Commands can have children, and those children can have their own aliases:

```py
@engine.command
def sell(amount: int):
    ...

@sell.subcommand(aliases=["alot"])
def all(*_):
    return sell(10000000)
```

This mirrors how established CLIs are structured (`git remote add`, `docker container ls`). Subcommands are defined directly on the parent with the same decorator pattern, so there's no separate command tree API to learn.

## String parsing built in

For interactive REPL-style tools, Invokify ships `string_to_args()`, a tokenizer that turns a raw input string into typed Python values. Quoted strings stay strings, bare numbers become `int` or `float`, and bracket syntax becomes nested lists:

```py
from invokify import string_to_args

string_to_args('greet "Jeff Bridges" 42 [1, 2, 3]')
# → ['greet', 'Jeff Bridges', 42, [1, 2, 3]]
```

Combined with an input loop, this is enough for a working command shell without `argparse` or `shlex`.

## Dependency injection

The `meta` helpers let commands declare what they need injected at call time:

- **`meta.require`** asks for the engine or the command object itself, or defines custom requirements your command can inspect.
- **`meta.inject`** passes in shared state, like a database connection or a game player object, without threading it through every parse call.
- **`meta.help`** attaches help text to a command for when you build your own help output.

This keeps command functions clean while still supporting patterns like command chaining, where one command parses and forwards to the next.

## How it's built

- **Minimal surface area.** Three modules: the engine and command decorators, a string tokenizer, and the public exports. No plugin system, no code generation.
- **Typed and dataclass-based.** Commands and the engine use `@dataclass(slots=True)` for a small memory footprint and clear structure.
- **Published on PyPI.** Install with `pip install invokify` or `poetry add invokify`. Requires Python 3.10+.

## Try it

```shell
pip install invokify
```
